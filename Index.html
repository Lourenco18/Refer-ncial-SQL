Referêncial SQL

Contextualização teorica

Structured Query Language (SQL), "linguagem de consulta estruturada", é uma linguagem de domínio específico desenvolvida para gerir dados relacionais num SGBD.

1. CREATE TABLE:

Cria uma nova tabela na base de dados.

CREATE TABLE [nome da tabela] ("coluna1" "tipo_de_dados_col1", "coluna2" "tipo_de_dados_col2"...);

➤ Exemplo

CREATE TABLE clientes (nome VARCHAR(50), idade INT(3));

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3)

2. INSERT INTO:

Adiciona novos registros a uma tabela.

2.1. VALUES:

Serve para especificar o valor das colunas

INSERT INTO [nome da tabela] (coluna1, coluna2) VALUES (valor1, valor2);

➤ Exemplo

INSERT INTO clientes (nome, idade) VALUES ('João', 30), ('Marta', 28), ('Tiago', 43);

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3)
João 30
Marta 28
Tiago 43

3. UPDATE:

Atualiza registros existentes em uma tabela.

UPDATE [nome da tabela] SET coluna1 = valor1, coluna2 = valor2;

➤ Exemplo

UPDATE clientes SET idade = 40 ;

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3)
João 40
Marta 40
Tiago 40

4. DELETE:

Remove registros de uma tabela.

DELETE FROM [nome da tabela] WHERE condition;

➤ Exemplo

DELETE FROM clientes WHERE nome = 'João';

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3)
Marta 40
Tiago 40

6. ALTER TABLE:

Modifica a estrutura de uma tabela existente (adiciona, remove ou modifica colunas).

6.1. ADD:

Adiciona uma coluna á tabela.

ALTER TABLE [nome da tabela] ADD [nome da coluna] tipo_de_dados;

➤ Exemplo

ALTER TABLE clientes ADD Nacionalidade VARCHAR(140), estado_civil VARCHAR(140);

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3) Nacionalidade VARCHAR(140) Estado Civil VARCHAR(140)
João 40
Marta 40
Tiago 40

6.2. DROP COLMN:

Elimina uma coluna da tabela.

ALTER TABLE [nome da tabela] DROP COLMN [nome da coluna];

➤ Exemplo

ALTER TABLE clientes DROP COLMN estado_civil;

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Nacionalidade VARCHAR(140)
João Portugal
Marta Espanha
Tiago Itália

6.3. RENAME COLUMN:

Muda o nome de uma coluna da tabela.

ALTER TABLE [nome da tabela] RENAME COLUMN [nome antigo] to [nome novo];

➤ Exemplo

ALTER TABLE clientes RENAME COLUMN Nacionalidade to Pais_origem;

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Pais_origem VARCHAR(140)
João Portugal
Marta Espanha
Tiago Itália

6.4. ALTER/MODIFY DATATYPE:

Muda o tipo de dados de uma coluna da tabela.

ALTER TABLE [nome da tabela] ALTER COLUMN [nome da coluna] tipo_de_dados;

ALTER TABLE [nome da tabela] MODIFY COLUMN [nome da coluna] tipo_de_dados;

ALTER TABLE [nome da tabela] MODIFY [nome da coluna] tipo_de_dados;

➤ Exemplo

ALTER TABLE clientes ALTER COLUMN Nome TEXT;

Resultado:

Tabela Clientes:

Nome TEXT Pais_origem VARCHAR(140)
João Portugal
Marta Espanha
Tiago Itália

7. DROP TABLE:

Remove uma tabela da base de dados.

DROP TABLE [nome da tabela];

➤ Exemplo

DROP TABLE clientes;

Resultado:

Tabela Base:

Nome VARCHAR(50) Idade INT(3) Nacionalidade VARCHAR(140) Estado Civil VARCHAR(140)
João 30 Portugal Casado
Marta 28 Espanha Solteiro
Tiago 43 Itália Solteiro

8. SELECT:

O comando SELECT permite selecionar colunas e linhas de uma tabela, pode-se adicionar uma a uma, ou, para selecionar todas de uma vez, usa-se o operador * .

8.1. FROM:

O comando FROM permite escolher a tabela a partir da qual os dados serão selecionados.

SELECT coluna1, coluna2 FROM [nome da tabela];

➤ Exemplo

SELECT nome, idade FROM clientes;

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3)
João 30
Marta 28
Tiago 43

9. WHERE:

Utilizado para filtrar registros com base em uma condição específica.

SELECT * FROM [nome da tabela] WHERE condition;

9.1. Condições do WHERE:

9.1.1 Igual (=):

Este operador realiza a operação de igualdade entre dois valores. Com ele verificamos se o valor à direita do sinal = é igual ao valor contido na coluna informada à esquerda.

SELECT * FROM [nome da tabela] WHERE dado = "..."

➤ Exemplo

SELECT * FROM clientes WHERE Estado Civil = "Solteiro";

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3) Nacionalidade VARCHAR(140) Estado Civil VARCHAR(140)
Marta 28 Espanha Solteiro
Tiago 43 Itália Solteiro

9.1.2 Diferente (!= ou <>):

Retorna registros onde o dado da coluna é diferente de um valor.

SELECT * FROM [nome da tabela] WHERE dado <> '...';

➤ Exemplo

SELECT * FROM clientes WHERE Estado Civil <> "Solteiro";

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3) Nacionalidade VARCHAR(140) Estado Civil VARCHAR(140)
João 30 Portugal Casado

9.1.3 Maior que (>), Menor que (<), Maior ou igual (>=) ou Menor ou igual (<=):

Retorna registros onde o dado de uma coluna é maior, menor, maior ou igual, menor ou igual a um valor.

SELECT * FROM [nome da tabela] WHERE dado ( < ou > ou <= ou >= ) [valor];

➤ Exemplo

SELECT * FROM clientes WHERE idade < "30";

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3) Nacionalidade VARCHAR(140) Estado Civil VARCHAR(140)
Marta 28 Espanha Solteiro

9.1.4 Entre (BETWEEN):

Retorna registros onde o dado de uma coluna esta entre dois valores.

SELECT * FROM [nome da tabela] WHERE dado BETWEEN X AND Y;

➤ Exemplo

SELECT * FROM clientes WHERE idade BETWEEN 29 AND 60;

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3) Nacionalidade VARCHAR(140) Estado Civil VARCHAR(140)
João 30 Portugal Casado
Tiago 43 Itália Solteiro

9.1.4 tipo de dados (LIKE / NOT LIKE):

Retorna registros onde o dado de uma coluna contem ou não contem um conjunto de palavras "...".

%... procura no fim da palavra
...% procura no início da palavra
%...% procura no meio da palavra

SELECT * FROM [nome da tabela] WHERE dado (LIKE ou NOT LIKE) "%...%";

➤ Exemplo

SELECT * FROM clientes WHERE nome LIKE "%o";

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3) Nacionalidade VARCHAR(140) Estado Civil VARCHAR(140)
João 30 Portugal Casado
Tiago 43 Itália Solteiro

9.1.5 Em (IN):

Com o operador IN podemos criar uma lista de valores que serão comparados com o valor na coluna. Dessa forma a linha a qual o valor avaliado pertence será ignorada exceto se qualquer um dos valores na lista corresponder ao valor da coluna.

SELECT * FROM [nome da tabela] WHERE dado IN (X, Y, ...);

➤ Exemplo

SELECT * FROM clientes WHERE idade IN (28, 43);

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3) Nacionalidade VARCHAR(140) Estado Civil VARCHAR(140)
Marta 28 Espanha Solteiro
Tiago 43 Itália Solteiro

9.2. Operadores - AND, OR, NOT:

9.2.1 AND:

SELECT * FROM [nome da tabela] WHERE condicao AND condicao;

➤ Exemplo

SELECT * FROM clientes WHERE Nacionalidade = 'Espanha' AND Nacionalidade = 'Itália';

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3) Nacionalidade VARCHAR(140) Estado Civil VARCHAR(140)
Marta 28 Espanha Solteiro
Tiago 43 Itália Solteiro

9.2.2 OR:

SELECT * FROM [nome da tabela] WHERE condicao OR condicao;

➤ Exemplo

SELECT * FROM clientes WHERE Nacionalidade = 'Espanha' OR Idade = '43';

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3) Nacionalidade VARCHAR(140) Estado Civil VARCHAR(140)
Marta 28 Espanha Solteiro
Tiago 43 Itália Solteiro

9.2.3 NOT:

SELECT * FROM [nome da tabela] WHERE NOT condicao;

➤ Exemplo

SELECT * FROM clientes WHERE NOT Idade = '43';

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3) Nacionalidade VARCHAR(140) Estado Civil VARCHAR(140)
João 30 Portugal Casado
Marta 28 Espanha Solteiro

10. ORDER BY:

Ordena os resultados da consulta por ordem crescente (ASC) ou decrescente (DESC).

SELECT * FROM [nome da tabela] ORDER BY coluna1 ASC;

➤ Exemplo

SELECT * FROM clientes ORDER BY idade ASC ;

Resultado:

Tabela Clientes:

Nome VARCHAR(50) Idade INT(3) Nacionalidade VARCHAR(140) Estado Civil VARCHAR(140)
Marta 28 Espanha Solteiro
João 30 Portugal Casado
Tiago 43 Itália Solteiro

Tabela Base:

id_produto INT Nome_produto VARCHAR(140) Tipo VARCHAR(140) Preco (INT)

2 Iphone 13 Telefone 1400€
3 Macbook pro Computador 4250€

11. Funções - COUNT(), AVG() e SUM() :

11.1. COUNT():

Número de linhas de uma tabela.

SELECT COUNT(coluna) FROM [nome da tabela];

➤ Exemplo

SELECT COUNT(id_produto) FROM Produtos;

Resultado: 3

11.2. AVG():

Média

SELECT AVG(coluna) FROM [nome da tabela];

➤ Exemplo

SELECT AVG(preco) FROM Produtos;

Resultado: 2283€

11.3. SUM():

Soma

SELECT SUM(coluna) FROM [nome da tabela];

➤ Exemplo

SELECT SUM(preco) FROM Produtos;

Resultado: 6850€

11.4. MAX():

Maior

SELECT MAX(coluna) FROM [nome da tabela];

➤ Exemplo

SELECT Max(preco) FROM Produtos;

Resultado: 4250€

11.4. MIN():

Menor

SELECT MIN(coluna) FROM [nome da tabela];

➤ Exemplo

SELECT MIN(preco) FROM Produtos;

Resultado: 1200€

12. GROUP BY:

Agrupa os resultados da consulta com base numa ou mais colunas.

SELECT coluna1 FROM [nome da tabela] GROUP BY coluna1;

SELECT COUNT(id_produtos) AS Quantidade FROM produtos GROUP BY tipo ;

13. INNER JOIN:

Combina linhas de duas ou mais tabelas com base em uma condição de junção.

[nome da tabela].campo - usado para destinguir os campos das tabelas

SELECT coluna1, coluna2 FROM tabela1 INNER JOIN tabela2 ON tabela1.coluna_chave = tabela2.coluna_chave;

➤ Exemplo

SELECT clientes.Nome, produtos.Nome_produto FROM clientes INNER JOIN produtos ON clientes.id_produto = produtos.id_produto;

Resultado:

Nome Nome_produto
João Iphone 12
Marta Macbook pro
Tiago Iphone 13

14. LEFT JOIN (ou LEFT OUTER JOIN):

Retorna todas as linhas da tabela à esquerda e as correspondentes da tabela à direita. As linhas não correspondentes da tabela à direita conterão valores NULL .

SELECT * FROM tabela1 LEFT JOIN tabela2 ON tabela1.coluna_chave = tabela2.coluna_chave;

➤ Exemplo

SELECT Clientes.Nome, Produtos.id_produto, Produtos.Nome_produto, Produtos.id_encomenda FROM Clientes LEFT JOIN Produtos ON Clientes.id_produto = Produtos.id_produto;

Resultado:

Nome VARCHAR(50) id_produto INT Nome_produto VARCHAR(140) id_encomenda (INT)
João 1 Iphone 12 2
Marta 3 Macbook pro 1
Tiago 2 Iphone 13
Daniel

Neste exemplo, todas as linhas da tabela "Clientes" são incluídas no resultado, juntamente com as linhas correspondentes da tabela "Produtos". Se não houver correspondência para o id_produto na tabela "Produtos", as colunas relacionadas a "Produtos" conterão valores nulos.

15. RIGHT JOIN (ou RIGHT OUTER JOIN):

Retorna todas as linhas da tabela à direita e as correspondentes da tabela à esquerda. As linhas não correspondentes da tabela à esquerda conterão valores NULL .

SELECT coluna1, coluna2 FROM tabela1 RIGHT JOIN tabela2 ON tabela1.coluna_chave = tabela2.coluna_chave;

➤ Exemplo

SELECT Clientes.Nome, Produtos.id_produto, Produtos.Nome_produto, Produtos.id_encomenda FROM Clientes RIGHT JOIN Produtos ON Clientes.id_produto = Produtos.id_produto;

Resultado:

Nome VARCHAR(50) id_produto INT Nome_produto VARCHAR(140) id_encomenda (INT)
João 1 Iphone 12 2
Marta 3 Macbook pro 1
Tiago 2 Iphone 13

Neste exemplo, todas as linhas da tabela "Produtos" são incluídas no resultado, juntamente com as linhas correspondentes da tabela "Clientes". Se não houver correspondência para o id_produto na tabela "Clientes", as colunas relacionadas a "Clientes" conterão valores nulos. Note que o "Daniel" não aparece no resultado, pois não há correspondência na tabela "Clientes" para o id_produto nulo na tabela "Produtos".

16. FULL OUTER JOIN:

Retorna todas as linhas quando há uma correspondência em uma das tabelas. As linhas não correspondentes em ambas as tabelas conterão valores NULL .

SELECT coluna1, coluna2 FROM tabela1 FULL OUTER JOIN tabela2 ON tabela1.coluna_chave = tabela2.coluna_chave;

➤ Exemplo

SELECT Clientes.Nome, Produtos.id_produto, Produtos.Nome_produto, Produtos.id_encomenda FROM Clientes FULL JOIN Produtos ON Clientes.id_produto = Produtos.id_produto;

Resultado:

Nome VARCHAR(50) id_produto INT Nome_produto VARCHAR(140) id_encomenda (INT)
João 1 Iphone 12 2
Marta 3 Macbook pro 1
Tiago 2 Iphone 13
Daniel

17. CROSS JOIN:

Retorna o produto cartesiano de duas tabelas, ou seja, combina cada linha da primeira tabela com cada linha da segunda tabela.

SELECT coluna1, coluna2 FROM tabela1 CROSS JOIN tabela2;

➤ Exemplo

SELECT Clientes.Nome, Produtos.id_produto, Produtos.Nome_produto, Produtos.id_encomenda FROM Clientes CROSS JOIN Produtos;

Resultado:

Nome VARCHAR(50) id_produto INT Nome_produto VARCHAR(140) id_encomenda (INT)
João 1 Iphone 12 2
João 2 Iphone 13
João 3 Macbook pro 1
Marta 1 Iphone 12 2
Marta 2 Iphone 13
Marta 3 Macbook pro 1
Tiago 1 Iphone 12 2
Tiago 2 Iphone 13
Tiago 3 Macbook pro 1
Daniel 1 Iphone 12 2
Daniel 2 Iphone 13
Daniel 3 Macbook pro 1

Neste exemplo, cada linha da tabela "Clientes" é combinada com cada linha da tabela "Produtos", produzindo todas as combinações possíveis entre os clientes e os produtos. O número total de linhas no resultado será o produto do número de linhas nas duas tabelas.

18. ÍNDICES:

Índices são estruturas de dados que melhoram a velocidade de operações numa tabela. Eles proporcionam uma forma rápida de encontrar dados em uma ou mais colunas de uma tabela.

CREATE INDEX [nome do índice] ON [nome da tabela] (coluna1, coluna2, ...);

➤ Exemplo

CREATE INDEX idx_id_produto ON Clientes (id_produto);

Isso cria um índice chamado idx_id_produto na coluna id_produto da tabela "Clientes".

19. VIEWS:

Views são consultas SQL armazenadas que podem ser tratadas como tabelas virtuais. Elas simplificam consultas complexas ao fornecer uma camada de abstração sobre tabelas subjacentes.

FROM Clientes c - isto serve para dar uma letra a tabela, simplificando a escrita do comando

CREATE VIEW [nome da view] AS SELECT coluna1, coluna2 FROM [nome da tabela] WHERE condition;

➤ Exemplo

CREATE VIEW vw_ClientesProdutos AS SELECT c.Nome AS Cliente_Nome, c.id_produto AS Cliente_id_produto, p.id_produto AS Produto_id_produto, p.Nome_produto AS Produto_Nome_produto, p.Preco AS Produto_Preco, FROM Clientes c JOIN Produtos p ON c.id_produto = p.id_produto;

Essa view (vw_ClientesProdutos) combina os dados das tabelas "Clientes" e "Produtos" com base na coluna id_produto.
